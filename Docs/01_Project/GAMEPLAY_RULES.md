# Gameplay Rules

## Player choice

Players should regularly choose between valuable opportunities. They should not be able to complete everything in one day.

## Time is a resource

Time pressure should create planning, cooperation and stories.

## Work must be active

Every profession needs multiple stations, decisions and opportunities.

## Progress must become visible

Important progression should change the world, not only numbers.

## Escape is not the only value

A mechanic should still be enjoyable even if the player is not currently escaping.

## Cooperation must have practical value

Avoid making every player independently capable of everything too early.

## Items communicate meaning clearly

- Background color communicates item category.
- Contraband uses a separate border or icon.
- There is no classic rarity-color system.
- Item value is understood through function, quality and context.

## Home workstations are bonuses, not replacements

A cell workstation removes routine and provides small recipes. Complex production remains tied to workplaces.

## Predictable schedule, unpredictable day

The schedule helps players plan. Events and risks keep days from feeling identical.

## Mechanics should create stories

Before implementing a mechanic, ask:

> What story will players tell after using it?

## Risk should be readable

Players should understand why an action is dangerous.

## Multiplayer world continues during UI

Opening inventory, containers, shops or crafting does not pause the world. Large UI windows block player movement and world interaction.
